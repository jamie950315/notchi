# Notchi 1.0.0

This is the first public release of Notchi, an animated notch companion for Claude Code.

Notchi lives in the MacBook notch and reacts to Claude Code activity in real-time. It connects through Claude Code's hook system — a shell script fires events over a Unix socket whenever Claude starts thinking, runs a tool, asks for permission, or finishes a task. Notchi turns those events into sprite animations, sounds, and a live activity feed, all tucked into the notch area.

## Hooks and Session Tracking

Notchi installs a hook script into `~/.claude/hooks/` on first launch. The script sends JSON events to a local Unix socket that Notchi listens on. Events include `session_start`, `session_end`, `thinking_start`, `thinking_end`, `tool_result`, `pre_tool_use`, `post_tool_use`, and `permission_request`.

Each Claude Code session gets its own tracked state. If you have multiple terminal windows running Claude Code simultaneously, Notchi tracks them independently with a session picker in the expanded panel.

## Sprite System

The character in the notch is a sprite-sheet-driven animation with five task states — idle, working, sleeping, compacting, and waiting — and four emotion overlays: neutral, happy, sad, and sob. The state machine maps hook events to the appropriate task and emotion combination. For example, a long thinking phase shows the working sprite, a permission request shows the waiting sprite, and session end transitions to sleeping.

Sprites bob gently in the notch. In the expanded panel, they sit on a grass island that you can tap to switch between sessions. Each session gets its own sprite positioned with depth sorting.

## Emotion Analysis

If you provide an Anthropic API key in Settings, Notchi runs user prompts through Claude Haiku to detect emotional tone. The result adjusts the sprite's emotion overlay — a frustrated message might shift the character toward sad, while a celebratory one triggers happy. Emotions build up gradually and decay over time rather than snapping between states.

## Expanded Panel

Click the notch to expand it into a panel showing:

- **Activity feed** — a timeline of hook events for the current session, with assistant messages rendered as markdown and user prompts displayed as chat bubbles
- **Permission details** — when Claude Code asks for permission, the panel shows what it wants to do and the available choices
- **Usage bar** — live Claude API quota tracking via OAuth, pinned to the bottom of the panel
- **Session stats** — event counts and timing for the active session

The panel has a pin button to keep it open and a mute toggle for sounds.

## Settings

The settings panel (gear icon in the expanded view) includes:

- **Screen selector** — pick which display to attach the notch overlay to, for multi-monitor setups
- **Launch at Login** — uses `SMAppService` to register as a login item
- **Hook status** — shows whether the Claude Code hook script is installed, with a button to reinstall
- **Usage connection** — connect to claude.ai for live quota tracking
- **Emotion analysis key** — paste an Anthropic API key to enable emotion detection
- **Sound picker** — choose notification sounds for completions and errors
- **Check for Updates** — manual update check (Sparkle framework)

## Sounds

Notchi plays a short chirp when a task completes and a different sound on errors or permission requests. Sounds respect a per-session cooldown to avoid rapid-fire notifications during busy sessions. Non-interactive sessions (`claude -p`) are detected and silenced automatically.

## System Requirements

- macOS 15.0 Sequoia or later
- MacBook with a notch (Apple Silicon)
- Claude Code installed with hooks support
